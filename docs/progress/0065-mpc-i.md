# 0065 — MPC-I：约束 + 积分器合体，NMPC 线收尾

**Commit**: `b150a06`  **Date**: 2026-04-23  **Milestone**: M9.5

## What / 做了什么

把 **M9.1 MPC**（约束 + 有限 horizon）和 **M9.3 LQI**（积分器消 bias）合成一个控制器。state augment 到 3 维：

```
z = [e_p, e_v, i]
A_z = [[1, dt, 0], [0, 1, 0], [dt, 0, 1]]
B_z = [dt²/2, dt, 0]
Q_z = diag(q_pos, q_vel, q_i),  R = r
```

i 是积分器维度，`i_{k+1} = i_k + e_p · dt`（嵌在 A_z 第三行）。

- `Mpc1dI<H>`：M9.1 Mpc1d 的 3 维版本，Hessian 仍是 H×H（只是动力学展开变成 3-dim），lin_from_z0 是 H×3
- `MpcI3dPositionController<H>`：3 轴 wrapper，warm buffer + integrator 都在内部
- `PositionController::MpcI` 变体 + `::mpc_i()` 构造器 —— 接入 M9.4 enum dispatch
- `dare_3x3` 私有 helper：augmented Riccati 用作终端代价

**7 新测试** 包括**关键 sanity check** `q_i = 0 ⇒ MPC-I ≡ M9.1 Mpc1d`（证明 3-dim 公式退化到 2-dim 是数值一致的）。Workspace **225 tests 全绿**。

SITL shootout realistic sim 新 case：MPC-I 和 PI+I / LQI 一起 < 0.6 m altitude err under 2 m/s 风 + drag。

## Why / 为什么这么做

### 为什么要合体 MPC 和 LQI

看 M9.1-M9.4 四路控制器的优劣矩阵：

| 能力 | PI | LQR | MPC | LQI | MPC-I |
|---|---|---|---|---|---|
| 无约束最优 | ❌ 手调 | ✅ | ✅ | ✅ | ✅ |
| 尊重 u_max | 事后 clamp | 事后 clamp | ✅ 预测 | 事后 clamp | ✅ 预测 |
| 消稳态 bias | ⚠️ 有 k_i | ❌ | ❌ | ✅ | ✅ |
| WCET 可预测 | ✅ | ✅ | ⚠️ iter | ✅ | ⚠️ iter |

MPC 解决了"约束感知"这件事，LQI 解决了"bias 鲁棒"那件事。**真机场景**里这两个问题同时存在：风/载荷/drag 带来常值 bias（要 integrator），而电池电压下降 / 单电机失效会让 u_max 变窄（要预测约束）。

单独的 MPC 或 LQI 各有半边功能 —— MPC-I 是完整的生产级控制器。

### 为什么积分器要"measurement-driven"而不是"prediction-driven"

在 MPC 的 QP 里，A_z 的第三行 `i_{k+1} = i_k + e_p_k · dt` 让 solver 预测未来 H 步内 integrator 怎么变。**QP 里的 integrator 是预测的**。

但 **真实 integrator 每 tick 在外部用真测量 e_p 更新**，不用 QP 预测的值：

```rust
// QP 解出 u_0
let u_0 = mpc.solve(z_0, &mut warm, iter);
// 外部 integrator 用 actual e_p 推一步
self.integrator += e_p * dt;
```

为什么不用 QP 预测的 i？
- **robust 到模型失配**：真动力学 ≠ 模型（drag、风、motor lag），QP 预测的 e_p_k 不对 → 预测的 i 也不对
- **measurement-based 反馈**：真测量的 e_p 反映实际偏差 → integrator 对真实 disturbance 积分，不对"模型以为会怎样"积分

这是标准 MPC-I 套路。Terminal cost 用 LQI Riccati（包含 integrator 维度）确保 horizon truncation 仍然有 stability guarantee。

### 为什么 `q_i = 0 ⇒ MPC-I ≡ M9.1 Mpc1d` 是关键 sanity

3-dim 动力学 DARE 出的 Hessian 应该在 q_i→0 时坍缩到 2-dim 的 Hessian（因为 i 轴 Q = 0 → P 的 i-block 退化）。具体：

- q_i = 0 → Q_z 只有 (1,1) 和 (2,2) 非零
- DARE 3×3 converged P 也应在 (3,3) 和 cross-terms 为 0
- Hessian 和 lin_from_z0 的 i-contribution 为 0
- 解出的 u_0 与 M9.1 Mpc1d 的 u_0（同 e_p, e_v）一致

这条**测试不通过就说明公式有 typo**。跑通了就说明整个 3-dim 推导是对的、矩阵 index 没错、DARE 数值稳定。

`< 0.05 m/s²` 的阈值是 f32 精度和 DARE 收敛 tolerance 的组合 —— 严格到能抓真 bug，松到不会被数值噪声弹。

### 为什么用 dare_3x3 私有 helper

`compute_lqi_gains` 是 M9.3 的公开 API，它内部做 3×3 DARE 但**只返回 gain**（k_p, k_v, k_i），不返回 P 矩阵。MPC-I 的终端代价需要 P 本身。

两个选择：
- 给 `compute_lqi_gains` 增加返回 P 的参数 → 公开 API 复杂化
- 抽出私有 `dare_3x3` 返回 P，`compute_lqi_gains` 和 `Mpc1dI::new` 各自调用并做自己需要的后处理

第二个更干净。代码微量重复（~30 行）但职责分明。

### 为什么 `MpcI3dPositionController.reset()` 同时清 warm 和 integrator

Mode 切换（比如 PI → MPC-I）或大 setpoint jump 时：
- warm buffers 里的预测轨迹是为"上一状态"算的 → 对新状态不优，可能拖慢收敛
- integrator 的累积值反映了"上一状态"的稳态 bias → 对新状态是噪声

一起清零确保干净起步。工程实践：当 GCS 发 MAVLink `PARAM_SET` 切 controller 时 call `reset()`。

### 为什么没在 enum 里做"从 MPC 无缝切换到 MPC-I"

技术上 MPC-I 是 MPC 的超集（q_i = 0 退化）。API 上却是**不同变体**：

- `PositionController::Mpc` 只有 warm buffer，没有 integrator state 可读
- `PositionController::MpcI` 两者都有
- 运行时从 Mpc 切换到 MpcI 需要初始化 integrator = 0 —— 简单但要管

如果 M10 要做参数化 controller（`PARAM_SET` 运行时切），可以让 enum 提供 `.promote_to_mpc_i()` 或 `.demote_to_mpc()` helper。现在不急。

### 为什么 shootout 三路（PI+I / LQI / MPC-I）都 < 0.6 m

整合器类控制器在 realistic sim 下的表现收敛到类似水平：约 15-20 秒收敛到稳态。MPC-I 的优势在**瞬态**（约束活跃时更平滑），在这个测试里没有制造瞬态紧约束，所以看不出 MPC-I 的独特优势。

要看 MPC-I 领先其他 integrator controllers：加一个 **"紧 u_max + 大 initial error + wind"** 场景。M10 candidate。

当前 shootout 只证"MPC-I 在既有场景里不掉队"。

## Follow-ups / 遗留

- **MPC-I vs MPC transient benchmark**：紧 u_max + wind + 大 initial error 场景，MPC-I 应该**更快收敛**因为预测约束 + 积分器同时起作用。
- **sim-hil shootout 加 MPC**（非 MPC-I）**在 realistic sim 下**会失败的 baseline，让 regression test 对比更具体。
- **Property test MPC-I**：随机 (weights, u_max, initial z_0)，PG 迭代后 Hessian 应该递减（单调下降不变量）。
- **WCET benchmark**：`criterion` bench MPC-I 每次 `.step()` 的 CPU 时间。Hessian 是 10×10，projected-gradient 25 iter。H7 @ 480 MHz 上预期 < 100 µs。
- **Accelerated projected gradient (Nesterov)**：iter 数减半，更多余量给增加 H 或更多 iter。
- **LQI / MPC-I 切换 transient**：从 LQI 切到 MPC-I 时积分器移交；单元测试模拟这个。
- **plan.md 四大创新点更新**：NMPC（创新点 2）正式落下，剩下 NN/RL (3) + Zenoh (4)。
- **`examples/controller_shootout.rs`**：非 test executable 把 5 路 controller（PI / LQR / MPC / LQI / MPC-I）表格化输出。
