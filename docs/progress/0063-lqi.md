# 0063 — LQI：把稳态偏差从 LQR 里清干净

**Commit**: `0e7d1c7`  **Date**: 2026-04-22  **Milestone**: M9.3

## What / 做了什么

M9.0 的 LQR 有个**致命弱点**：它对"模型正确"的前提打满分。一旦真机有阻力/风/payload 变化 —— 任何 **constant disturbance** 进来，LQR 会稳在一个**稳态偏差**上不肯回到目标。

经典补丁：augmented state LQI。状态扩到 `z = [e_p, e_v, i]`，3×3 DARE 出三个增益 `[k_p, k_v, k_i]`。控制律 `u = -K · z` 自动在 integrator 里积起 bias，最终 k_i · i 正好对消 disturbance。

API（和 LQR/MPC 对称）：
```rust
let mut lqi = Lqi3dPositionController::new(
    weights_xy, weights_z, dt, max_accel, integrator_max
)?;
let att = lqi.step(&setpoint, pos, vel, mass);
```

- `LqiWeights { q_pos, q_vel, q_i, r }` — 4 个有物理意义的参数
- `compute_lqi_gains(weights, dt) -> LqiAxisGains { k_p, k_v, k_i }` — 解 3×3 DARE
- `Lqi3dPositionController` — 3 积分器状态 + 反 windup + 共享 last-mile

**6 新单元测试 + 1 新 SITL**（realistic sim 下 drag + 2 m/s 风，15 秒内 altitude 偏差 < 0.3 m、horizontal < 1 m），workspace **213 tests 全绿**。

## Why / 为什么这么做

### 为什么 q_i = 0 必须严格等价 LQR

测试 `lqi_with_zero_qi_matches_lqr` 是 **sanity check 的护航测试**。它验证：

- 3×3 DARE 的数学展开正确
- A_z / B_z / Q_z 矩阵构造没有 typo
- DARE 迭代收敛位置和 2×2 版本一致
- 增益提取的 row/col 索引对

如果这个测试跪了，其他 LQI 测试的结果就**不可信** —— 可能"测试通过"只是因为 bug 的量级还没显化。能证明 "q_i=0 ⇔ LQR" 就确认了整个 LQI 实现的数学 backbone 稳。

### 为什么 integrator 积的是 e_p 而不是 e_v

两个都能 kill steady-state bias，但选择不同：

- **积 e_p**（我们做的）：standard LQI。对应位置-目标的**累计偏离时间** → 直接表征"飞机欠/超过目标多久了"。物理直觉清晰。
- **积 e_v**（PI cascade 里的 `integrator_vel`）：velocity error 的积分。数学上 ≈ 位置偏差（积分消掉导数），但近似性依赖初值 e_p(0) = 0。

对 LQI 而言，积 e_p 是经典式，DARE 闭式得到 [k_p, k_v, k_i]。积 e_v 得再变换，不优雅。保留 PI cascade 的现有实现，LQI 走标准路径，**不相互污染**。

### 为什么 anti-windup 用"conditional integration"而不是 back-calculation

两种主流反-windup：

1. **Conditional integration**：command 饱和时暂停积分。
   ```rust
   if !saturated { integrator += e_p * dt; }
   ```
2. **Back-calculation**：积分量按"溢出量"反向扣除。
   ```rust
   let overflow = u_unsat - u_sat;
   integrator -= gain_bc * overflow * dt;
   ```

PI cascade 用方案 1（M3.2 实现），LQI 用同一方案 —— **保持一致**便于操作员理解行为。

Back-calculation 更快恢复，但增加一个 tuning 参数 (gain_bc) 和潜在的震荡模式。LQI 的 integrator_max clamp 已经提供一层 fallback。

### 为什么 integrator_max = 5

默认 disturbance 估计：`max_accel_compensable = k_i * integrator_max`。我们希望能对消 `max_accel_compensable ≈ 2-4 m/s²`（典型风 + drag 的压力）。

k_i 典型值 ~0.5-1（权重决定），integrator_max = 5 给出 ~2.5-5 m/s² 补偿能力，比 **hover thrust 的 25% 级别**，不会让 vehicle 因 integrator wind-up 飞成无主。

如果应用场景有极端 bias（比如 单电机失效 → 持续 5 m/s² 不对称 yaw torque），可以调高。

### 为什么 realistic SITL 阈值这么宽（0.3m / 1m）

对比 M3.2 PI：
- realistic sim + drag + wind → 0.6 m altitude 稳态误差
- 已经是"能接受的量级"但不够严格

LQI 在同场景：
- 0.3 m altitude（测试阈值），实测可能更小，随 q_i 权重 tightness

Horizontal 1 m 宽是因为：
- xy 方向风扰直接施加，需要 integrator 先积起来再对消
- 15 秒收敛时间 + 位置环 response time constant 决定
- 紧一点可能要 20-30 秒

PI cascade + wind_ff (M3.3) 能做到 < 0.5 m horizontal，但那依赖 EKF 有好的 wind estimate。LQI 不需要 wind estimate，**纯靠 integrator 自己发现 bias**，更鲁棒。

### 为什么不把 LQI 直接接进 `outer_step`

和 MPC 一样的问题：`outer_step` PI-locked、糅了 landing/takeoff/RTL override 逻辑。要接 LQI 得：
1. `RateLoopConfig::pos_controller: enum`
2. `FlightState::controller_state` 按 variant 存不同内部状态
3. outer_step dispatch

这是架构层 refactor，M9.4 scope。M9.3 只负责"LQI 数学正确 + 闭环能飞"。

### 三路控制器对比总结

| Controller | 优势场景 | 限制 |
|---|---|---|
| **PI cascade** (M3.2) | 常规稳态有 wind_ff、轻量、可解释 | 手调 gains、无最优性保证 |
| **LQR** (M9.0) | 模型已知、最优性证明、纯数学 | 对 drag/wind 有 steady bias |
| **MPC** (M9.2) | 约束最优、预测未来 | 每 tick 求解 iter 成本、无 integrator |
| **LQI** (M9.3) | 自动消 bias、无 wind estimate 依赖 | 无约束、比 MPC 慢响应 |

最强组合（M9.5 候选）：**MPC-I** = MPC + 积分状态 = 约束感知 + bias 清除。augment 成 3H 状态空间的 QP，Hessian 大一圈但结构没变。

## Follow-ups / 遗留

- **M9.4 controller selector 进 app-copter**：用 enum 让 outer_step 可选 PI / LQR / MPC / LQI。
- **M9.5 MPC-I**（augmented-state MPC）：把 LQI 的整合状态和 MPC 的约束意识合体。
- **WindFF + LQI 组合**：EKF 给出 wind 估计，feed-forward 到 LQI 的 setpoint.accel_ned，进一步减小 integrator wind-up 时间。
- **Property test**：随机 (weights, disturbance, initial state)，closed-loop 应在可证明时间内收敛到 bias=0。
- **Integrator leak / forget factor**：长期静止偏差 + 温度漂移可能让 i 累积到无意义。加 leak term `i *= 0.99` 或 Kalman-filter 式 decay。
- **Benchmark `controller_shootout`**：写 `sim-hil/examples/controller_shootout.rs`，跑同样场景四路控制器，输出 RMS tracking error 表格。
- **Kani 证 integrator bounded**：给定 clamp = integrator_max，无论输入 sequence 如何，∥i∥ ≤ integrator_max 恒成立。
- **温度补偿**：用 IMU temperature 映射 drag coef，dynamic LQI 重算。
