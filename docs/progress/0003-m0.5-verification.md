# 0003 — M0.5 验证全绿

**Commit**: `4c23282`  **Date**: 2026-04-20  **Milestone**: M0.5（M0 收官 + 进 M1 前的闸门）

## What / 做了什么

在容器里把 M0 脚手架验证了三遍：

```
cargo check  --workspace             ok   (12 crate，21 s)
cargo clippy --workspace -D warnings ok
cargo test   --workspace             ok   (0 tests — 骨架阶段)
cargo run    -p app-copter --release 打印任务图
```

触发出的两个问题 + 修复：

1. **Cargo volume 权限**：Docker named volume `/opt/rust/cargo/registry` 首次挂载时归 root。`dev` 用户（UID 1000）无法写入。**修法**：在 `docker/entrypoint.sh` 加了 chown 步骤，用户每次启动容器时自动修（利用 Dockerfile 给 dev 的 NOPASSWD sudo）。
2. **Clippy lints 过激**：原来 workspace 启用了 `clippy::pedantic` + `clippy::cargo` 组；后者逼每个 crate 都得有 `readme/keywords/categories` 字段，对 private monorepo 无意义，每个 crate 报 36 个 error。**修法**：`Cargo.toml` 里重写 lints policy。

## Why / 为什么这么做

### Volume ownership 的根因

Docker volume 的初始 ownership 是由 **写入者的** UID 决定的，但 volume mount 会在目录创建时先显示 root。如果镜像 build 阶段 chown 过，mount 盖上空 volume 后又变 root。entrypoint 里动态修是最稳的。

### Clippy 分组的取舍

| 分组 | 是否启用 | 理由 |
|------|---------|------|
| `clippy::correctness`（默认 deny） | 是 | 真 bug |
| `clippy::suspicious`（默认 warn） | 是 | 可能 bug |
| `clippy::style`（默认 warn） | 是 | 风格 |
| `clippy::complexity`（默认 warn） | 是 | 可读性 |
| `clippy::perf`（默认 warn） | 是 | 性能 |
| `clippy::pedantic`（默认 allow） | **否** | 过严、假阳性多、`must_use_candidate` 之类会刷屏 |
| `clippy::cargo`（默认 allow） | **否** | 目标是 crates.io 发布者，不适合内部 monorepo |
| `clippy::nursery`（默认 allow） | **否** | 实验性，会误判 |

保留下来的项目级 deny：`panic` / `unwrap_used` / `expect_used` / `indexing_slicing`——对应 CLAUDE.md 的"firmware 里绝不 panic"。这些会在 M1+ 写真正代码时发挥作用。

## How it's verified / 怎么验证的

```bash
# 容器内
docker exec docker-dev-1 bash -c 'cd /workspace && cargo check --workspace'
docker exec docker-dev-1 bash -c 'cd /workspace && cargo clippy --workspace --all-targets -- -D warnings'
docker exec docker-dev-1 bash -c 'cd /workspace && cargo test --workspace'
docker exec docker-dev-1 bash -c 'cd /workspace && cargo run -p app-copter --release'
```

期望输出：全部 `Finished` + `test result: ok`；`app-copter` 打 task graph（imu_sample / rate_loop_indi / ekf_update / nmpc_outer 等）。
